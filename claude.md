# AgriFlow Backend Development Context

AgriFlow is a fertilizer subsidy distribution management system built to manage stock flow and subsidy redemption from Government → Distributor → Retailer → Farmer.

Farmer is NOT a system user.

Farmer data already exists externally (e-RDKK assumption).

Financial transactions are OUT OF SCOPE.

# Business Rules

## BR-01 Farmer Validation

PetaniId:

must exist

must be 16 digits

must belong to retailer

Else:

reject.

---

## BR-02 Stock Validation

Shipment:

stock >= requested

Else reject.

Redemption:

retailer stock >= requested

Else reject.

---

## BR-03 Quota Validation

Requested qty:

must not exceed quota.

Else reject.

---

## BR-04 Shipment Receiving

received == sent

→ Diterima

received != sent

→ Tidak Sesuai

Generate notification.

---

## BR-05 Atomic Transactions

Use Prisma transaction.

Shipment:

reduce stock

insert shipment

insert history

insert log

commit together.

Redemption:

reduce stock

reduce quota

insert redemption

insert log

commit together.

Receiving:

update shipment

update stock

insert notification

commit together.
